# `git_scythe` Test Scenarios

This document provides a summary of the test scenarios for the main application logic, which handles user interaction and orchestrates calls to the `git_utils` package.

| Scenario                  | Description                                                                                                                            | Motivation                                                                                                                                      | Utility                                                                                                                                                                 | Test Location                                                                                              | Test Type |
| :------------------------ | :------------------------------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------- | :-------- |
| **Successful Deletion**   | Verifies the end-to-end flow where a list of branches is displayed, the user confirms with "Y", and the deletion logic is called.         | This is the primary "happy path" for the application. It ensures that a user's confirmation correctly triggers the branch deletion process.         | It validates that the core functionality of the tool works as expected from the user's perspective, providing confidence in the main application loop.                  | `TestRun/successful_deletion`                                                                              | Unit      |
| **User Cancels**          | Verifies that if the user enters "n" at the confirmation prompt, the deletion logic is not called and an "Operation cancelled" message is shown. | A user must always have a clear and respected way to abort a destructive operation. This test ensures the cancellation path works correctly.      | It proves that the tool is safe and respects the user's decision to cancel, preventing accidental deletions.                                                          | `TestRun/user_cancels`                                                                                     | Unit      |
| **No Branches to Delete** | Verifies that when `GetMergedBranches` returns an empty list, the tool prints a "No merged branches" message and exits gracefully.       | This is a common and expected state for a clean repository. The tool should provide clear, informative feedback rather than showing an empty prompt. | It ensures a good user experience by handling the "nothing to do" case explicitly, which prevents user confusion.                                                      | `TestRun/no_branches_to_delete`                                                                            | Unit      |
| **Error Fetching Branches** | Verifies that if the initial call to `GetMergedBranches` returns an error, the error is correctly propagated and the program exits.      | This test handles foundational errors, such as not being in a Git repository. The tool must fail gracefully if it cannot get the initial list of branches. | It ensures the application is robust against environmental errors and will not proceed to later stages if the initial, critical step of fetching branches fails. | `TestRun/error_fetching_branches`                                                                          | Unit      |
