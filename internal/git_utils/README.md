# `git_utils` Test Scenarios

This document provides a summary of the test scenarios for the `git_utils` package, which provides an abstraction layer over the Git command-line tool.

| Scenario                 | Description                                                                                                                                                                                    | Motivation                                                                                                                                                                                             | Utility                                                                                                                                                                  | Test Location                                                                                             | Test Type   | Dependencies | Execution Command                                                                                             |
| :----------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------- | :---------- | :----------- | :------------------------------------------------------------------------------------------------------------ |
| **Get Default Branch**   | Verifies that the `GetDefaultBranch` function can successfully parse the output of `git rev-parse` to find the repository's default branch name (e.g., `main` or `master`). It also handles non-`origin` remote names. | The default branch is a critical piece of information needed for many other Git operations, such as determining which branches have been merged. This test ensures we can reliably identify it.        | It allows other parts of the application to be independent of the specific default branch name, making the tool more portable across different repositories.             | `TestGetDefaultBranch/Success`<br>`TestGetDefaultBranch/Error`<br>`TestGetDefaultBranch/non-origin_remote` | Integration | `git` CLI    | `go test -run ^TestGetDefaultBranch`                                                                          |
| **Get Merged Branches**  | Tests the primary workflow of piping `git branch --merged` to `grep` to produce a clean list of merged branches. It also tests the failure case where the default branch cannot be determined. | This is a core feature of the `git-scythe` tool. The test ensures that the command pipeline is constructed correctly and that the output is parsed into a clean, usable list of strings.               | It validates that the application can correctly identify which branches are safe to delete, which is fundamental to the tool's purpose.                                  | `TestGetMergedBranches/successful_execution`                                                              | Integration | `git` CLI    | `go test -run ^TestGetMergedBranches/successful_execution$`                                                   |
| **No Merged Branches**   | Tests the case where no branches have been merged into the default branch.                                                                                                                     | A common state in new repositories. The function should handle this gracefully by returning an empty slice, not `nil` or an error.                                                              | Ensures the tool behaves predictably and doesn't report an error when it's simply a new or clean repository.                                                              | `TestGetMergedBranches/no_merged_branches`                                                                | Integration | `git` CLI    | `go test -run ^TestGetMergedBranches/no_merged_branches$`                                                     |
| **Unusual Branch Names** | Verifies that branch names containing special characters (e.g., slashes, spaces, or names that resemble command-line flags like `--version`) are parsed correctly.                               | Branch names with special characters are common. This test ensures that the `grep` command and string parsing logic don't misinterpret these characters, preventing potential bugs or security issues. | Guarantees that the tool will work reliably in repositories that follow common naming conventions for feature, hotfix, or release branches.                               | `TestGetMergedBranches/unusual_branch_names`<br>`TestGetMergedBranches/branch_names_with_spaces`<br>`TestGetMergedBranches/branch_name_conflicts_with_flag` | Integration | `git` CLI    | `go test -run ^TestGetMergedBranches/unusual_branch_names$`                                                   |
| **Detached HEAD State**  | Ensures that `GetMergedBranches` correctly ignores the `* (HEAD detached at ...)` line that appears when the repository is in a detached HEAD state.                                            | This is a common state when checking out commits directly. The tool should not treat the "detached HEAD" line as a branch, which would cause errors or incorrect behavior.                     | Prevents the tool from failing or producing incorrect output when run from a detached HEAD, which is a common workflow for developers reviewing old commits.             | `TestGetMergedBranches/detached_HEAD_state`                                                               | Integration | `git` CLI    | `go test -run ^TestGetMerged-Branches/detached_HEAD_state$`                                                   |
| **Git Command Resilience** | Ensures the tool handles errors gracefully when `git` commands fail. This includes scenarios where `git` is not in the system's PATH, a command is run outside of a valid Git repository, or the repository is empty. | The tool must be resilient to environmental issues. These tests guarantee that the program will fail with a clear, understandable error message instead of crashing. | This provides a better user experience by giving clear feedback when the tool is used incorrectly, preventing confusion and making it easier to diagnose setup problems. | `TestGetDefaultBranch/Git_not_in_path`<br>`TestGetMergedBranches/error_in_getting_default_branch`<br>`TestGetDefaultBranch/Empty_repository` | Integration | `git` CLI    | `go test -run ^TestGetDefaultBranch/Git_not_in_path$`<br>`go test -run ^TestGetMergedBranches/error_in_getting_default_branch$` |
