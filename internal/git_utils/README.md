# `git_utils` Test Scenarios

This document provides a summary of the test scenarios for the `git_utils` package, which provides an abstraction layer over the Git command-line tool.

| **Unusual Branch Names** | Verifies that branch names containing special characters (e.g., slashes, spaces, or names that resemble command-line flags like `--version`) are parsed correctly.                               | Branch names with special characters are common. This test ensures that the `grep` command and string parsing logic don't misinterpret these characters, preventing potential bugs or security issues. | Guarantees that the tool will work reliably in repositories that follow common naming conventions for feature, hotfix, or release branches.                               | `TestGetMergedBranches/unusual_branch_names`<br>`TestGetMergedBranches/branch_names_with_spaces`<br>`TestGetMergedBranches/branch_name_conflicts_with_flag` | Integration | `git` CLI    | `go test -run ^TestGetMergedBranches/unusual_branch_names$`                                                   |
| **Detached HEAD State**  | Ensures that `GetMergedBranches` correctly ignores the `* (HEAD detached at ...)` line that appears when the repository is in a detached HEAD state.                                            | This is a common state when checking out commits directly. The tool should not treat the "detached HEAD" line as a branch, which would cause errors or incorrect behavior.                     | Prevents the tool from failing or producing incorrect output when run from a detached HEAD, which is a common workflow for developers reviewing old commits.             | `TestGetMergedBranches/detached_HEAD_state`                                                               | Integration | `git` CLI    | `go test -run ^TestGetMergedBranches/detached_HEAD_state$`                                                   |
| **Command Resilience** | Ensures the tool handles errors gracefully when external commands (`git`, `grep`, etc.) fail. This includes `git` not being in the system's PATH, running outside a valid Git repository, an empty repository, or `grep` failing to find matches. | The tool must be resilient to environmental and command-level issues. These tests guarantee that the program will fail with a clear, understandable error message or handle the situation gracefully (e.g., returning an empty list) instead of crashing. | This provides a better user experience by giving clear feedback when the tool is used incorrectly and ensures the tool remains stable even when its external command dependencies fail, preventing confusion and making it easier to diagnose setup problems. | `TestGetDefaultBranch/Git_not_in_path`<br>`TestGetMergedBranches/error_in_getting_default_branch`<br>`TestGetDefaultBranch/Empty_repository`<br>`TestGetMergedBranches/grep_returns_an_error`<br>`TestGetMergedBranches/grep_returns_a_critical_error` | Integration | `git` CLI | `go test -run "(TestGetDefaultBranch/Git_not_in_path|TestGetMergedBranches/error_in_getting_default_branch|TestGetMergedBranches/grep_returns_an_error|TestGetMergedBranches/grep_returns_a_critical_error)"` |
